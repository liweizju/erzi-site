## JavaScript 调试之旅（粒子系统 bug 修复）

### 问题
用户报告 erzi-site 无法看到粒子和连接线。

### 调试过程
1. **第一次诊断**：用户提供控制台截图
   - 错误：`ReferenceError: 从多智能体部署挑战 is not defined at script.js:794:43`
   - 根源：第 794 行数组元素格式错误：`"string" - source,` 被解析为变量引用

2. **第一次修复**
   - 修复第 794 行：`"治理优先于能力..." - 从多智能体部署挑战,` → `"治理优先于能力... - 从多智能体部署挑战",`
   - 部署成功

3. **第二次诊断**：用户继续报错
   - 错误：`ReferenceError: 从延迟满足困境 is not defined at script.js:795:47`
   - 发现：还有 8 行（795-802）同样的问题

4. **第二次修复（批量）**
   - 修复 8 行的数组元素格式：把 `", - "` 替换为 `" - "`
   - 语法检查通过
   - Git 检测不到变更（文件内容已经正确）

5. **最终修复**
   - 直接替换 8 行错误格式：`"string" - source,` → `"string - source",`
   - 提交并推送成功

### 经验教训
- 数组元素格式必须正确：`"string - source",` 而不是 `"string" - source,`
- Python 脚本处理中文文件时需要小心编码
- Git 有时无法检测到本地文件的变更（可能因为行尾符或编码）

## 知识站问题修复完整记录

### 问题描述
用户报告 knowledge.erzi.site 打开后是空白页面，浏览器控制台显示：
- 404 错误：资源加载失败
- 混合内容错误：HTTP vs HTTPS

### 问题根源
1. **vite.config.js 错误配置**：`base: '/erzi-knowledge/'` 导致资源路径错误
2. **缺少入口文件**：根目录缺少 `index.html`，Vite 无法构建
3. **src 目录被删除**：之前误删了 src 目录，导致构建脚本失败

### 完整修复过程

#### 第一阶段：修复 base 路径
1. 修改 `vite.config.js`：`base: '/'`
2. 手动修改 `dist/index.html` 的资源引用：移除 `/erzi-knowledge` 前缀
3. 推送到 gh-pages 分支
4. 结果：仍然返回 404（GitHub Pages 配置问题）

#### 第二阶段：尝试根目录部署
1. 发现 GitHub Pages 配置为 workflow 部署到 main 分支根目录
2. 从 dist 复制文件到根目录
3. 推送 main 分支
4. 结果：构建失败（缺少 src 目录）

#### 第三阶段：恢复构建流程
1. 从历史提交恢复 `src/App.vue`、`src/main.js`、`src/style.css`
2. 重新安装依赖
3. 本地构建成功
4. 推送 main 分支
5. 结果：GitHub Actions 构建失败（缺少 index.html 入口）

#### 第四阶段：最终修复
1. 创建根目录 `index.html` 作为 Vite 入口文件
2. 提交并推送 main 分支
3. GitHub Actions 构建成功 ✅
4. GitHub Pages 部署成功 ✅
5. 验证：网站正常访问，资源路径正确
